# Meridian — AI-Native News Intelligence

> Depth-first news coverage generated by an autonomous 4-stage AI editorial pipeline. No human editors. Economist-level rigour as the target standard.

**Live demo:** https://meridian-news.vercel.app

---

## What it is

Meridian is a fully autonomous AI news platform. Every article — from topic selection to final prose — is produced by a structured pipeline with no human editorial input. The output targets the analytical depth of The Economist and Reuters, not the volume-first approach of content farms.

Each article includes:
- Sharp, non-clickbait headline and subheadline
- Structured sections with meaningful H2 headings
- Data callouts with specific statistics
- Pull quotes as analytical insight
- Mandatory **What we know / What we don't know** section
- Transparent AI pipeline trace

## The 4-stage pipeline

```
Stage 1: Topic Discovery
  └── Simulates real-world signal scoring (connect to NewsAPI/GDELT in production)

Stage 2: Angle Expansion
  └── Dedicated Claude call → structured editorial brief (JSON)
      Returns: primaryAngle, keyTensions, sections, dataNeeds, whatWeKnow, whatWeDontKnow

Stage 3: Article Generation
  └── Claude generates full article from the structured brief
      House style enforced via system prompt
      Returns: headline, subheadline, byline, body (HTML), digest

Stage 4: Storage + Publication
  └── Article assembled with metadata, stored, routed to /article/[id]
```

- Node.js 18+

```bash
# 1. Clone / download this project
cd meridian

# 2. Install dependencies

# 3. Add your API key
echo "ANTHROPIC_API_KEY=sk-ant-your-key-here" > .env.local

npm run dev
```

## Deploy to Vercel (5 minutes)

### Option A — Vercel CLI (fastest)

```bash
# Install Vercel CLI
npm i -g vercel

# Deploy from project root
cd meridian
vercel

# Follow prompts, then add env var:
vercel env add ANTHROPIC_API_KEY
# Paste your API key when prompted

# Deploy to production
vercel --prod
```

Your URL: `https://meridian-[random].vercel.app`

### Option B — GitHub + Vercel Dashboard

```bash
# 1. Create a GitHub repo
git init
git add .
git commit -m "Initial Meridian deploy"
git remote add origin https://github.com/YOUR_USERNAME/meridian-news.git
git push -u origin main

# 2. Go to https://vercel.com/new
# 3. Import your GitHub repo
# 4. Add environment variable:
#    Name:  ANTHROPIC_API_KEY
#    Value: sk-ant-your-key-here
# 5. Click Deploy
```

Vercel auto-detects Next.js. Build takes ~60 seconds. Done.

## Customisation

### Swap in-memory store for a real database

In `lib/store.js`, replace the in-memory array with:

**Upstash Redis (recommended for Vercel):**
```bash
npm install @upstash/redis
```

```js
import { Redis } from '@upstash/redis'
const redis = new Redis({ url: process.env.UPSTASH_URL, token: process.env.UPSTASH_TOKEN })

export async function getArticles() {
  const ids = await redis.lrange('article_ids', 0, 49)
  return Promise.all(ids.map(id => redis.get(`article:${id}`)))
}
export async function addArticle(article) {
  await redis.set(`article:${article.id}`, article)
  await redis.lpush('article_ids', article.id)
}
```

### Connect real news APIs (Stage 1)

In `lib/pipeline.js`, replace `fetchTrendingTopics()` with:

```js
// NewsAPI
const res = await fetch(`https://newsapi.org/v2/top-headlines?apiKey=${process.env.NEWS_API_KEY}&pageSize=10`)
const { articles } = await res.json()
return articles.map(a => ({ topic: a.title, category: categorise(a), signal: a.source.name }))
```
